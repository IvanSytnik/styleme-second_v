Это **третий этап** — впервые в v3 есть **рабочее приложение**, через которое ты можешь загрузить фото и получить AI-результат от Replicate.

| Слой | Что |
|---|---|
| **Дизайн-система** | Полная палитра, дизайн-токены, светлая + тёмная темы, переключатель System/Light/Dark |

## 4️⃣ Главная проверка — реальный E2E

Открой http://localhost:3000.

### 4.1 — Стартовый экран

Должен увидеть:

```
StyleMe  [Free 3/3]  [☀ ⌥ ☾]
─────────────────────────────────
   Try a new hairstyle (с градиентом на "hairstyle")

   ┌─────────────────────────┐
   │  ⬆                       │
   │  Drop a photo or click   │
   │  to choose               │
   │  JPEG, PNG or WebP       │
   └─────────────────────────┘

   [📷 Take a photo]

   Your photo stays on your device until you start a generation...
```

### 4.2 — Тест переключателя темы

В шапке справа: три иконки `☀ ⌥ ☾`. Покликай:

- **☀ (Light)** — белый фон, тёмный текст
- **☾ (Dark)** — тёмный фон, светлый текст, акцент чуть розовее
- **⌥ (System)** — следует настройке твоей ОС

Состояние сохраняется в localStorage между перезагрузками страницы. ✅

### 4.3 — Тест загрузки фото

Перетащи фото на dropzone **или** кликни по нему.

- На <2 секунды появится `Preparing your photo…`
- Затем перейдёт на **экран catalog**

⚠️ Если тащишь фото >10 МБ — браузер успешно сожмёт до 1024px и отправит ~200 КБ. Это работа `resizeImage()` через Canvas API.

### 4.4 — Тест выбора стиля

На экране catalog:

- Табы **Women / Men** переключают список (20 + 20)
- Кликни на любую карточку — она подсветится розовым (border + soft bg)
- Кнопка **`Generate ✨`** внизу справа активируется
- В footer внизу: круглое превью твоего фото + кнопка «Back»

Нажми **Generate**.

### 4.5 — Тест processing (это где платится деньги)

⚠️ **Этот шаг реально тратит твой Replicate-бюджет** (~$0.04 за вызов).

Появится экран:
- Твоё фото с shimmer-анимацией
- Заголовок `Creating your new look`
- Название выбранного стиля
- Honest indeterminate progress bar (бежит туда-сюда, без fake %)
- Подпись `Usually takes 15–30 seconds…`

Через 15–30 секунд → **экран result**.

### 4.6 — Тест result

- Side-by-side: твоё фото (Before) | AI-результат (After с розовым лейблом)
- **Кнопка `Share`** — Web Share API (на мобильных откроет шит «поделиться», на desktop скопирует URL)
- **Кнопка `Download ⬇`** — сохранит результат как JPEG
- **Try another style** — назад в catalog (фото сохранится)
- **Use another photo** — полный reset, на upload

### 4.7 — Тест error path: QUOTA_EXCEEDED

Сгенерируй 3 раза подряд (израсходуй все 3 бесплатных кредита). На 4-й попытке вместо processing увидишь:

```
No credits left
You've used all your free generations for today.
Watch an ad to earn more, or come back tomorrow.

[← Back]
```

Это правильно — server-side квота защищает тебя от overspend. ✅

### 4.8 — Тест header balance

После каждой генерации **в шапке** обновляется счётчик: `Free 3/3` → `Free 2/3` → `Free 1/3` → `Free 0/3`. Это значит React Query кэш правильно инвалидируется после mutation.

## ⚠️ Типичные проблемы

### Тема мигает (Flash) при загрузке страницы

Этого **не должно быть** — `next-themes` инжектит inline script в `<head>` через `suppressHydrationWarning`. Если видишь flash — проверь что в `layout.tsx` у `<html>` стоит `suppressHydrationWarning`.

### `npm install` падает на `next-themes`

Это новый пакет — возможно npm кэш битый. Попробуй:
```bash
rm -rf node_modules apps/*/node_modules package-lock.json
npm install
```

### При загрузке фото на iPhone Safari фото поворачивается на 90°

Это **EXIF orientation** — мой код использует `createImageBitmap` с `imageOrientation: 'from-image'`, который правильно учитывает EXIF на всех современных браузерах. Если проблема осталась — пришли версию Safari, разберёмся.

### Кнопка `Take a photo` на desktop открывает обычный file picker

Это **правильно**. Атрибут `capture="user"` действует только на мобильных. На desktop fallback — обычный picker.

### Балансы в шапке не обновляются после генерации

Открой DevTools → Network → проверь что после `/api/transform` запроса идёт `/api/billing/balance`. Если нет — React Query кэш не инвалидируется. Сообщи мне.

### `Generation failed` — `UPSTREAM_FAILED`

Replicate отказался обработать запрос. Самые частые причины:
- Кончились деньги на Replicate-аккаунте → пополни баланс
- Replicate временно недоступен → попробуй через минуту
- Размер картинки после resize > 2 МБ → не должно случаться (max — около 200 КБ), но если случилось — сообщи

### `RATE_LIMITED` после одной попытки

Не должно случаться (limit 10/час/user). Возможно у тебя несколько вкладок дёргают API параллельно. Закрой лишние вкладки и подожди минуту.
